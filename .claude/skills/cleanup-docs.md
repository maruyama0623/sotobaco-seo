# /cleanup-docs — ドキュメントFAQ整理スキル

## 概要
`docs/` 内のFAQ・学習データの重複や類似エントリを検出し、社長の承認を得て統合・整理する。

## 実行手順

### 1. 対象ドキュメントの読み込み
- `docs/sotobaco-portal.md` のFAQセクションを読み込む
- `docs/btone.md` のFAQセクションを読み込む（存在する場合）
- 他に `docs/*.md` でFAQセクションがあればそれも対象にする

### 2. 重複・類似エントリの検出
以下の基準で重複を判定する:
- **完全重複**: 質問文と回答内容が同一のエントリ
- **類似重複**: 質問文が異なるが、回答の手順・内容が実質的に同じエントリ
- **包含関係**: 一方のFAQが他方のFAQの内容を完全に含んでいるエントリ

### 3. 統合案の提示
- 検出した重複・類似エントリを一覧で社長に報告する
- 各グループについて統合案（マージ後のエントリ内容）を提示する
- **社長の承認なしに統合・削除は行わない**

### 4. 承認後の統合実行
- 社長が承認したグループのみ統合を実行する
- FAQテーブルのフォーマットを統一する（番号の振り直し等）
- 変更前後のdiffを報告する

### 5. フォーマット統一
- FAQテーブルの列構成を統一（`| # | 質問 | 回答 |`）
- 番号を連番に振り直す
- テーブルヘッダー・区切り行の書式を統一する

## 注意事項
- 学習システム（support-api Worker）が自動追加したエントリと、手動追加したエントリが混在している可能性がある
- 統合時は情報の欠落がないよう、より詳細な回答を残す方針で統合する
- `git pull` を実行してから作業を開始する（学習システムによる最新の変更を取得するため）
