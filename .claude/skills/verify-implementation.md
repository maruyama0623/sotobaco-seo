# /verify-implementation — 実装後の動作確認スキル

## 概要
開発作業の完了後に、Playwright MCPを使って実際にアクセスし、実装した機能を一通り試してエラーがないか確認する。

## 実行手順

### 1. devサーバーの起動確認
- `npm run dev` 等でローカル開発サーバーが起動しているか確認
- 起動していなければ Bash でバックグラウンド起動する
- サーバーが ready になるまで待つ

### 2. 基本表示の確認
- `mcp__playwright__browser_navigate` で対象ページにアクセス
- `mcp__playwright__browser_snapshot` でページ構造を確認
- `mcp__playwright__browser_take_screenshot` でスクリーンショットを撮影
- 表示崩れ・エラーがないか確認

### 3. 実装した機能の動作確認
- 実装内容に応じて以下を確認:
  - **リンク遷移**: クリックして遷移先が正しいか
  - **フォーム**: 入力→送信→結果表示の一連のフロー
  - **ボタン・インタラクション**: クリック→期待する動作が起こるか
  - **データ表示**: 正しいデータが表示されているか
  - **エラーハンドリング**: 不正入力時に適切なエラーが出るか

### 4. レスポンシブ確認
- `mcp__playwright__browser_resize` でビューポートを変更して確認:
  - デスクトップ: 1280x800
  - タブレット: 768x1024
  - スマートフォン: 375x667
- 各サイズでレイアウト崩れがないかスクリーンショットで確認

### 5. コンソールエラーの確認
- `mcp__playwright__browser_console_messages` でエラー・警告を確認
- JavaScript エラーがあれば修正する

### 6. 結果報告
- **問題なし**: 確認した内容と結果を社長に報告
- **問題あり**: 問題を修正してから再度確認し、完了報告

## 注意事項
- Playwright MCPツールの直接呼び出しのみ使用する
- JavaScript/Python等でのブラウザ操作コード実行は禁止
- エラー時は回避策を探さず、エラーメッセージをそのまま報告する
