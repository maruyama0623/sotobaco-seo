# /check-docs-version — Context7 ドキュメント鮮度チェックスキル

## 概要
Next.js関連の開発開始時に、Context7のドキュメントがプロジェクトのバージョンをカバーしているか確認する。

## 実行手順

### 1. プロジェクトのバージョン確認
- `package.json` から Next.js のバージョンを確認する

### 2. Context7 の対応バージョン確認
- `mcp__context7__resolve-library-id` で Next.js を検索
- 利用可能なバージョン一覧を取得する

### 3. 判定
- **カバーされている場合**: 「Context7のドキュメントはプロジェクトのNext.js vX.X.Xに対応しています」と報告し、開発を進める
- **カバーされていない場合**: 以下を社長に報告:
  - プロジェクトのバージョン
  - Context7で利用可能な最新バージョン
  - 「公式ドキュメントの直接参照を併用します」の旨

### 4. 推奨ソースの選定
バージョンが対応している場合、用途に応じてソースを使い分ける:
- **APIリファレンス・コード例**: `/websites/nextjs`（公式サイトベース）
- **ソースコード寄りの詳細**: `/vercel/next.js/vX.X.X`（GitHub）
