# /write-article — SEO記事執筆スキル

## 概要
kintone関連市場向けのSEO記事を、調査から執筆・仕上げまで一貫して行う。

## 実行手順

### 1. 事前準備
- `docs/blog-media.md` を読み込む（記事制作方針・スタイル・回遊構造・記事一覧）
- `docs/company.md` を読み込む（共通注意事項・URL）
- 対象サービスの docs を読み込む（`docs/sotobaco-portal.md` / `docs/btone.md` 等）

### 2. 競合・参考メディア調査
- `docs/blog-media.md` の参考メディア一覧から、同テーマの記事をWebFetchで取得
- 扱っている情報・視点・見出し構成・情報の深さを把握する
- **YouTube動画も調査する**: WebSearchで関連動画を探し、YouTube Transcript MCP（`get_transcript`, `lang: "ja"`）で上位3〜5本の文字起こしを取得。動画でしか語られていないユーザーの悩み・実践ノウハウを記事に活かす
- 差別化ポイントを特定する（競合が書いていない切り口、自社ならではの解決策）
- kintone公式（kintone.cybozu.co.jp / cybozu.dev）で機能仕様の裏付けを取る

### 3. 構成設計
- 記事種別を確認（共感 / 方法 / 機能 / 7分）
- `docs/blog-media.md` の基本構成テンプレートに従って見出し構成を作成
- メインキーワード・関連キーワードを設定
- 文字数目安: 共感3,000〜4,000字 / 方法4,000〜5,000字 / 機能3,000字 / 7分2,500字
- 社長に構成案を確認してもらう

### 4. 執筆
- トヨクモブログスタイルに従う（ですます調、箇条書き多用、比較表活用）
- タイトル: `前半|後半` 形式（前半15文字以内 / 後半20文字以内）
- 導入文: 読者の悩みに共感 → ベネフィット提示（200〜300字）
- 「この記事でわかること」を箇条書き3〜5点で設置
- 自社サービスは解決策の最後に「一番おすすめ」として紹介
- CTA: 記事中盤（解決策として自然に）+ 記事末尾（まとめ後）
- まとめ: 箇条書き3〜5点 + CTA（フリープラン誘導）

### 5. SEO・内部リンク
- メインキーワード: タイトル・最初のH2・導入文・まとめに含める
- 関連キーワード: 自然に散りばめる
- メタディスクリプション: 120〜130字
- 回遊構造に従い内部リンクを設置（共感→方法→機能→7分→LP）

### 6. 画像配置
- H2/H3の画像配置ルールに従う（入れるなら全部、入れないなら全部入れない）
- 「まとめ」直下には画像を入れない
- Altテキストは `images/README.md` からコピー
- 画像がない場合は `images/no_image.jpg` をプレースホルダーとして使用

### 7. 仕上げ
- `docs/blog-media.md` の記事一覧に追記する
- 社長に完成報告（記事の要点・文字数・CTA配置箇所）

## 鉄則
- ノウハウを全部書かない（「何が必要か」は伝え、「どうやるか」の細部はサービスに委ねる）
- ただの解説記事にしない（次のアクションへの動線を必ずつなぐ）
- ページ単体で完結させない（内部リンクで回遊させる）
- 必ず「最短の導線」を作る（まとめ直前に一文でCTAへ誘導）
- 価格はすべて税抜表示
