# /respond-inquiry — 問い合わせ回答作成スキル

## 概要
顧客からの問い合わせに対して、関連ドキュメントを参照し、PII配慮した回答文を作成する。全サービス（ソトバコポータル・Btone・今後追加されるサービス）に対応。

## 実行手順

### 1. 問い合わせ内容の把握
- 問い合わせ内容からサービス・カテゴリを判別する
- 曖昧な場合は社長に確認する

### 2. 関連ドキュメント参照
- `docs/company.md` を参照（共通情報・URL）
- サービスに応じた docs を参照:
  - ソトバコポータル → `docs/sotobaco-portal.md`
  - Btone → `docs/btone.md`
  - 今後追加されるサービス → 該当する `docs/*.md`
- FAQ学習データ（各docsのFAQセクション）を確認し、過去の類似問い合わせがないか調べる

### 3. 回答文の作成
- **宛名**: 「会社名＋ご担当者様」を使用（個人名はAIに送信しない）
- **構成**:
  1. お問い合わせへのお礼
  2. 質問への回答（具体的・簡潔に）
  3. 補足情報（関連する操作ガイドのURLなど）
  4. 締めの挨拶
- **曖昧な問い合わせの場合**: 推測で回答せず、3〜5つの候補を番号付きで提示。お客様が番号をコピーして返信するだけで済む形式にする。最後に「上記以外」選択肢を必ず入れる

### 4. PII配慮チェック
- 回答文に個人情報（メールアドレス・氏名・電話番号）が不必要に含まれていないか確認
- ログやSlackに出力する場合はPIIを除去

### 5. 社長に確認
- 回答文の案を社長に提示する
- 修正指示があれば反映する

## 注意事項
- 価格に言及する場合は必ず税抜表示
- 回答文に noreply@sotobaco.com から送信する旨がある場合、「返信してください」等の文言を含めない
