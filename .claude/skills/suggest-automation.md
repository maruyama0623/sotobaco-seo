# /suggest-automation — 自動化提案スキル

## 概要
claude-memの過去セッション記録を分析し、繰り返し発生している作業パターンを検出して、スキル化・フック化の提案を行う。

## 実行手順

### 1. 過去の依頼パターンを収集
- claude-memの `search` で以下のカテゴリを横断検索する:
  - `type: "session-request"` — 過去の依頼内容
  - `type: "feature"` — 実装した機能
  - `type: "change"` — 行った変更
- 直近1〜2ヶ月の記録を対象にする

### 2. 繰り返しパターンの検出
以下の基準で繰り返しを判定する:
- **同じ種類の作業が3回以上**: 例）FAQ整理、記事執筆、問い合わせ対応
- **同じ手順を毎回踏んでいる**: 例）git pull → docs読み込み → 編集 → デプロイ
- **同じファイル群を繰り返し操作**: 例）docs/*.md の更新パターン

### 3. 自動化候補の分類
検出したパターンを以下に分類する:

| 分類 | 条件 | 例 |
|------|------|-----|
| スキル化 | 社長の判断・入力が必要な定型作業 | 記事執筆、競合調査、ドキュメント整理 |
| フック化 | 判断不要で自動実行できる処理 | git pull、フォーマット整形、ファイル保護 |
| 対応不要 | 1〜2回しか発生していない、または既にスキル/フック化済み | — |

### 4. 提案の報告
社長に以下の形式で報告する:

**繰り返し検出されたパターン**（頻度が高い順）:

1. **パターン名**（検出回数: N回）
   - 過去の実行例: セッション#xxx, #yyy
   - 提案: スキル化 or フック化
   - 理由: なぜ自動化すべきか

2. ...

**既存スキル/フックとの重複チェック**:
- `.claude/skills/*.md` と `.claude/hooks/*.sh` を確認し、既に対応済みのパターンは除外する

### 5. 承認後の実装
- 社長が承認したパターンのみ、スキルまたはフックとして実装する
- 実装後は `docs/worker.md` 等の該当ドキュメントも更新する

## 注意事項
- 提案のみ行い、承認なしにスキル/フックを作成しない
- 過去1〜2回しか発生していないパターンは提案しない（ノイズになるため）
- 既存のスキル・フックと機能が重複しないか必ず確認する
