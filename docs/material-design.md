# 資料作成ルール

HTML → PDF方式でスライド資料を作成する際のデザインルールを定義する。

---

## 基本方針

- **余白を作らない**: スライド内のコンテンツは画面全体を使い切るようにレイアウトする。ヘッダーとフッターの間の空間に対してコンテンツが小さく見えないよう、要素のサイズ・パディング・ギャップを調整する
- **ブランドカラーを活用**: `#35b597`（brand）/ `#E5F8F2`（brand-light）/ `#2a9a7f`（brand-dark）を基調とする
- **フォント**: Noto Sans JP を基本フォントとし、`font-feature-settings: "pkna"` で日本語プロポーショナルカーニング適用

---

## スライドサイズ

- 幅: 1280px
- 高さ: 720px
- アスペクト比: 16:9（横向き）

---

## レイアウトルール

### 余白の最小化

- ヘッダー（見出し + ロゴ）の下からフッターの上までの領域を **コンテンツ領域** とする
- コンテンツ領域は、要素のパディング・ギャップ・フォントサイズを調整して **できる限り埋める**
- グリッドやカードを使う場合は、カードの高さを `flex: 1` や `height` 指定で均等に広げ、コンテンツ領域全体を使う
- テキスト量が少ないスライドでは、フォントサイズを大きくする・画像を大きく表示する・装飾要素を追加するなどして空間を埋める

### スライド構成

- **ヘッダー**: 見出し（左）+ ロゴ（右）、高さ約70px
- **フッター**: ロゴ（左）+ コピーライト（右）、高さ約40px
- **コンテンツ領域**: 残りの約610pxを最大限活用する
- **左右パディング**: 50px（デフォルト）

### グリッド・カードレイアウト

- カードの高さは明示的に指定するか、`flex: 1` で親要素に合わせて伸縮させる
- 2x4グリッド（8項目）の場合: 2行は均等に高さを分割し、各行がコンテンツ領域の半分を占める
- 3カラムの場合: 各カラムが同じ高さになるよう `align-items: stretch` を使用する

## PDF化手順

1. ブラウザ（Chrome推奨）でHTMLを開く
2. `Cmd + P`（印刷）
3. 送信先: 「PDFとして保存」
4. 用紙サイズ: カスタム（1280 x 720）またはA4横
5. 余白: なし
6. 背景のグラフィック: ON
